# @mxrpl/dapp

Interactive dApp (standalone, Aliit phase): connect 1AM → prove eligibility on Midnight in-wallet →
request the XRPL credential from the gateway service → accept + use it. The heavy browser Midnight
stack (WASM, proving-key assets) is isolated here.

## Build prerequisite — compile the contract first
The dApp serves the circuit's **ZK proving keys as static assets** (`FetchZkConfigProvider`). Those keys
(`managed/keys` + `managed/zkir`, ~37 MB) are **generated by the Compact compiler**, not committed
(gitignored). So the build has a prerequisite, exactly like the canonical bboard-ui:

```sh
# 1) compile the contract — REQUIRES the Compact toolchain (WSL/Linux), same as compile/prove/e2e
npm run compile -w @mxrpl/private-credential-gateway-contract
# 2) build the dApp (typecheck -> vite build -> copy keys/zkir into dist/)
npm run build -w @mxrpl/dapp
```

If step 1 hasn't run, the build fails with a clear message (not a raw ENOENT). **This is a WSL/Linux
build** (the proving backend isn't available on bare Windows); the repo's unit tests (`node --test`) do
**not** need the keys and run anywhere. To deploy, build in WSL and publish `dist/` (a static bundle),
or run the compile step in a Linux CI that has the toolchain.

## Dev
```sh
npm run dev -w @mxrpl/dapp            # vite dev server
npm run typecheck -w @mxrpl/dapp      # tsc --noEmit
```
Env (`.env` / Vite): `VITE_NETWORK_ID`, `VITE_GATEWAY_URL` (the gateway service), `VITE_CONTRACT_ADDRESS`.

## Layout
- `src/midnight/providers.ts` — 1AM connector (dapp-connector-api v4) + browser providers (FetchZkConfig,
  httpClientProof via the wallet-supplied prover URI, indexer, in-memory private state).
- `src/midnight/gateway-api.ts` — join + `proveEligibility` (in-wallet proof).
- `src/lib/gateway-client.ts` — typed `POST /issue-credential` to the gateway service.
- `src/App.tsx` — connect + status shell (the full flow UI builds on this).

See `../../docs/SPIKE_BROWSER_PROVING.md` for the proving architecture and `docs/PRIVACY_BOUNDARY.md`
for the trust posture.
